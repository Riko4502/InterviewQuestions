**Что такое ECMAScript? В чём отличие от JavaScript?**

ECMAScript (ES) - это стандарт языка программирования, на котором базируется JavaScript. Он определяет синтаксис и функциональность языка, включая типы данных, операторы, объекты и методы. JavaScript является реализацией ECMAScript и предоставляет дополнительные функции и объекты, не определенные в стандарте ECMAScript.

**Разница между let, const и var?**

Переменные, объявленные с помощью const, нельзя переназначить (изменить ссылку на другой объект). Однако, если переменная содержит ссылку на изменяемый объект, такой как массив или объект, то содержимое этого объекта может быть изменено.

**Что такое временная мёртвая зона (temporal dead zone)?**

Временная мёртвая зона (temporal dead zone) - это фаза внутри блока или функции, где переменная существует, но не может быть доступна или использована до тех пор, пока не будет выполнено ее объявление с помощью let или const. Попытка доступа к переменной во время временной мертвой зоны приведет к ошибке ReferenceError.

**Разница между Rest и Spread операторами?**

Rest оператор (также называемый остаточным оператором) обозначается тремя точками (...). Он используется для объединения аргументов функции в массив или для деструктуризации массива или объекта.
Spread оператор также обозначается тремя точками (...). Он используется для распыления (spread) элементов массива или объекта, например, для передачи аргументов функции в виде отдельных значений или для создания нового массива/объекта на основе существующего с добавлением новых элементов.

**Что такое деструктуризация?**

Деструктуризация - это синтаксическая конструкция в JavaScript, позволяющая извлекать значения из массивов или объектов и присваивать их переменным с помощью синтаксиса, подобного шаблону.

**Для чего используется цикл for…of?**

Цикл for...of используется для перебора элементов итерируемых объектов, таких как массивы, строки, Map, Set и других, в порядке возрастания индекса или в соответствии с пользовательским итератором.

**Что такое шаблонные литералы (Template Literals)?**

Шаблонные литералы (Template Literals) - это расширение синтаксиса строк в JavaScript, которое позволяет встраивать выражения внутри строковых литералов с помощью синтаксиса **${выражение}**. Они также поддерживают многострочные строки без необходимости использования экранирования.

**Что такое Set, Map, WeakMap и WeakSet?**

- Set представляет собой упорядоченный набор уникальных значений любого типа.
- Map представляет собой коллекцию пар ключ-значение, где ключи и значения могут быть любого типа.
- WeakMap аналогичен Map, но он позволяет только объекты быть ключами, а также имеет слабую ссылку на ключи, что позволяет сборщику мусора удалять их, если они больше не используются.
- WeakSet представляет собой коллекцию уникальных объектов и также имеет слабые ссылки на объекты.

**Разница между обычными функциями и стрелочными?**

- Стрелочные функции имеют более короткий синтаксис и автоматическую привязку контекста (this) к окружающей функции.
У стрелочных функций отсутствует свой собственный контекст (this), и они не могут быть использованы в качестве конструкторов (не имеют своего собственного объекта this).
- Обычные функции имеют свою собственную область видимости переменных и доступ к объекту this, который определяется контекстом вызова функции.

**Разница между методом Object.freeze() и const?**

- Метод Object.freeze() используется для замораживания объекта, делая его неизменяемым. Замороженный объект не может быть изменен путем добавления, удаления или изменения свойств. Однако это не делает неизменяемыми свойства вложенных объектов или элементы массивов, содержащихся в замороженном объекте.
- Ключевое слово const используется для объявления переменной с неизменяемым (константным) значением. Оно ограничивает переназначение переменной, но не делает неизменяемыми свойства объекта или элементы массива, на которые она ссылается. Значение переменной, объявленной с const, не может быть изменено, но сам объект или массив могут быть изменены.

**Что такое итераторы?**

Итераторы - это объекты, которые предоставляют последовательный доступ к элементам коллекции (например, массиву), один элемент за раз. Итераторы имеют метод next(), который возвращает объект со свойствами value (значение текущего элемента) и done (логическое значение, указывающее на завершение итерации). Цикл for...of использует итераторы для перебора элементов коллекции.

**Что такое генераторы? Когда стоит использовать генераторы?**

Генераторы - это специальный тип функций в JavaScript, которые могут быть приостановлены и возобновлены во время выполнения. Они используют ключевое слово yield для возврата промежуточного значения и сохранения своего состояния. Генераторы могут производить последовательность значений с помощью итератора, делая их мощными для работы с большими объемами данных, асинхронными операциями и ленивой загрузкой данных. Они могут быть использованы, когда требуется обработка значений по мере необходимости, а не сразу во время вызова функции.

**Что такое ES6 модули?**

ES6 модули - это стандартный механизм модульной системы, представленный в ECMAScript 6 (ES6). 
Они позволяют разделять код на отдельные модули, имеющие свою область видимости, и экспортировать и импортировать значения между модулями. 
ES6 модули поддерживают именованный экспорт и импорт, экспорт по умолчанию и циклические зависимости между модулями.

**Что такое символ (Symbol) в ES6?**

Символ (Symbol) в ES6 - это новый тип данных, представляющий уникальный и неизменяемый идентификатор. Каждый символ уникален и может быть использован в качестве ключа свойства объекта. Они полезны, когда требуется создание уникальных имен свойств объекта, чтобы избежать конфликтов с другими свойствами или именами.

**Для чего используется метод .includes()?**

Метод .includes() используется для проверки наличия заданного значения в строке или массиве. Он возвращает логическое значение true, если значение найдено, и false в противном случае.

**Для чего используется метод .getOwnPropertyDescriptors()?**

Метод .getOwnPropertyDescriptors() используется для получения всех дескрипторов свойств объекта (таких как значения, геттеры, сеттеры, наличие свойства writable и т. д.) в виде объекта с ключами, соответствующими именам свойств объекта.

**Расскажите о методах .keys(), .values(), .entries()?**

Методы .keys(), .values() и .entries() используются для итерации по свойствам объекта и возвращают итераторы:

- **.keys()** возвращает итератор, перебирающий все ключи объекта.
- **.values()** возвращает итератор, перебирающий все значения свойств объекта.
- **.entries()** возвращает итератор, перебирающий массивы [ключ, значение] для каждого свойства объекта.

**Для чего используется метод .fromEntries()?**

Метод .fromEntries() используется для преобразования списка пар ключ-значение (представленного в виде массива массивов или итерируемого объекта) в новый объект. Каждый внутренний массив или итерируемый объект должен содержать два элемента: ключ и значение.

**Для чего используются методы .flat() и .flatMap()?**

Методы .flat() и .flatMap() используются для работы с массивами:

- **.flat()** позволяет "сплющивать" (flatten) вложенные массивы на указанную глубину. Он создает новый массив, в котором все элементы вложенных массивов расположены на одном уровне.
- **.flatMap()** комбинирует операции flat() и map(). Он сначала применяет функцию к каждому элементу массива, а затем "сплющивает" результат в новый массив.

**Для чего используются методы .padStart() и .padEnd()?**

Методы .padStart() и .padEnd() используются для добавления заданного количества символов (обычно пробелов) в начало или конец строки, чтобы достичь заданной длины строки. Эти методы полезны, когда требуется выровнять текст по определенной ширине.

**Для чего используются методы .startsWith() и .endsWith()?**

Методы .startsWith() и .endsWith() используются для проверки, начинается ли строка с определенного подстроки или заканчивается ли строка на определенную подстроку. Они возвращают логическое значение true, если условие выполняется, и false в противном случае.

**Как в JavaScript удалять пробельные символы в начале и в конце строки?**

Для удаления пробельных символов в начале и конце строки в JavaScript можно использовать методы .trim(), .trimStart() (ранее .trimLeft()) и .trimEnd() (ранее .trimRight()). Метод .trim() удаляет все пробельные символы с обоих концов строки, а .trimStart() и .trimEnd() удаляют пробельные символы только с начала или конца строки соответственно.

**Расскажите об операторе Optional Chaining (?.)?**

Оператор Optional Chaining (?.) в JavaScript используется для обращения к свойствам или вызову методов объекта, если эти свойства или методы существуют (не равны undefined или null). Если объект или свойство равны undefined или null, оператор ?. вернет undefined, не вызывая ошибку.

**Для чего используется метод .replaceAll()?**

Метод .replaceAll() используется для замены всех вхождений подстроки в строке на новую подстроку. Раньше для этой цели использовался метод .replace(), но он заменял только первое вхождение подстроки. Метод .replaceAll() позволяет заменить все вхождения одновременно.

**Что такое оператор логического присваивания?**

Оператор логического присваивания (Logical Assignment Operator) - это комбинация оператора присваивания и логического оператора, который позволяет одновременно проверить условие и присвоить значение переменной, если условие выполняется. Примеры таких операторов: **||=**, **&&=**, **??=**. Например, **x ||= y** означает, что если x имеет ложное значение (**false**, **null**, **undefined**, **0**, **пустая строка**), тогда x будет присвоено значение **y**.

**Как увеличить читаемость больших чисел?**

Для увеличения читаемости больших чисел в JavaScript можно использовать разделитель групп разрядов, который облегчает восприятие и чтение чисел. Разделитель групп разрядов представляется символом нижнего подчеркивания _. Например, число 1000000 может быть записано как 1_000_000, что делает его более читаемым.

**Что такое приватные аксессоры?**

Приватные аксессоры (Private Accessors) - это новая функциональность, введенная в языке JavaScript, позволяющая объявлять приватные свойства объекта и ограничивать доступ к ним. 
Они представлены синтаксическими конструкциями **#** перед именем свойства, например, **#property**. Приватные аксессоры обеспечивают контролируемый доступ к свойствам объекта и защищают их от несанкционированного доступа или изменения.

**Разница между ES6-классами и конструкторами функций?**

- ES6-классы представляют собой синтаксический сахар над конструкторами функций и прототипным наследованием в JavaScript. Они предоставляют более понятный и удобный синтаксис для создания объектов и работы с наследованием.
- Конструкторы функций - это функции, которые могут быть использованы для создания новых объектов с помощью ключевого слова new. Они определяют свойства и методы объекта с помощью свойства **this**.

**Что такое оператор нулевого слияния (??)?**

Оператор нулевого слияния (Nullish Coalescing Operator) - это оператор, который возвращает первый операнд, если он не равен null или undefined. В противном случае, если первый операнд равен null или undefined, возвращается второй операнд. Оператор нулевого слияния представляется двумя вопросительными знаками **??**. Например, **x ?? y** вернет **x**, если **x не равно null** или **undefined**, и **y** в противном случае.

**В чём отличие оператора нулевого слияния (??) и оператора “ИЛИ” (||)?**

Оператор нулевого слияния (??) отличается от оператора "ИЛИ" (||) в том, что оператор "ИЛИ" выполняет проверку на ложное значение (false, null, undefined, 0, пустая строка), тогда как оператор нулевого слияния выполняет проверку только на null или undefined. Таким образом, оператор нулевого слияния возвращает второй операнд только в случае, когда первый операнд равен null или undefined.

**Назовите основные методы и свойства работы с коллекцией Map?**

Некоторые основные методы и свойства работы с коллекцией Map:

- .set(key, value): добавляет элемент с указанным ключом и значением в Map.
- .get(key): возвращает значение, связанное с указанным ключом.
- .has(key): проверяет наличие элемента с указанным ключом в Map.
- .delete(key): удаляет элемент с указанным ключом из Map.
- .size: свойство, возвращающее количество элементов в Map.
- .keys(): возвращает итератор, перебирающий все ключи Map.
- .values(): возвращает итератор, перебирающий все значения Map.
- .entries(): возвращает итератор, перебирающий массивы [ключ, значение] для каждого элемента Map.

**Назовите основные методы и свойства работы с коллекцией Set?**

Некоторые основные методы и свойства работы с коллекцией Set:

- .add(value): добавляет значение в Set.
- .has(value): проверяет наличие значения в Set.
- .delete(value): удаляет значение из Set.
- .size: свойство, возвращающее количество элементов в Set.
- .keys(): возвращает итератор, перебирающий все значения Set.
- .values(): возвращает итератор, перебирающий все значения Set.
- .entries(): возвращает итератор, перебирающий массивы [значение, значение] для каждого элемента Set.

**Как осуществить перебор элементов в коллекциях Map и Set?**

Перебор элементов в коллекциях Map и Set можно осуществить с помощью цикла for...of. 
Например:

```
const myMap = new Map();
myMap.set("key1", "value1");
myMap.set("key2", "value2");

for (const [key, value] of myMap) {
  console.log(key, value);
}

const mySet = new Set();
mySet.add("value1");
mySet.add("value2");

for (const value of mySet) {
  console.log(value);
}
```
