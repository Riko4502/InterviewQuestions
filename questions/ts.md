<h3>
  <span>TypeScript:</span>
</h3>

**Что такое TypeScript?**

TypeScript - это язык программирования, который является надмножеством JavaScript. Он предоставляет статическую типизацию, то есть позволяет указывать типы данных переменных, параметров функций и возвращаемых значений. TypeScript компилируется в обычный JavaScript, что позволяет его использование везде, где применяется JavaScript.

**Основные компоненты TypeScript?**

возможностями, такими как типы, интерфейсы и модули.
1. Типы данных: TypeScript позволяет явно указывать типы данных для переменных, параметров функций, свойств объектов и т.д.
2. Интерфейсы: TypeScript поддерживает создание интерфейсов, которые описывают структуру объекта или класса и могут использоваться для проверки соответствия типов.
3. Классы и наследование: TypeScript поддерживает объектно-ориентированное программирование с помощью классов, наследования, инкапсуляции и полиморфизма.
4. Модули: TypeScript имеет встроенную поддержку модулей, что позволяет организовывать код в отдельные файлы и модули, упрощая повторное использование кода и поддержку больших проектов.

**Назовите особенности TypeScript?**

1. Статическая типизация: TypeScript позволяет обнаруживать и предотвращать ошибки типизации на этапе компиляции, что улучшает надежность кода и облегчает его сопровождение.
2. Поддержка ECMAScript: TypeScript поддерживает последние версии стандарта ECMAScript (JavaScript), а также предоставляет возможность использовать новые функции и возможности JavaScript до их полной поддержки браузерами.
3. Инструменты разработки: TypeScript интегрируется с различными инструментами разработки, такими как среды разработки (например, Visual Studio Code), что облегчает разработку, отладку и рефакторинг кода.
4. Расширяемость: TypeScript можно расширять с помощью декораторов, что позволяет добавлять новые функции и возможности к существующим объектам и классам.

**Плюсы использования TypeScript?**

1. Улучшенная надежность кода: Статическая типизация позволяет обнаруживать ошибки на этапе компиляции, что уменьшает количество ошибок времени выполнения.
2. Улучшенная поддержка IDE: TypeScript предоставляет богатую информацию о типах, что позволяет IDE предлагать подсказки, автодополнение и проверку типов, улучшая процесс разработки.
3. Читаемость и понятность кода: Явное указание типов делает код более понятным и самодокументирующимся, особенно в больших проектах с командной разработкой.
Поддержка новых возможностей языка: TypeScript поддерживает последние возможности ECMAScript и позволяет использовать их до полной поддержки в браузерах.

**Минусы использования TypeScript?**

1. Дополнительный этап компиляции: TypeScript требует компиляции в JavaScript перед запуском, что может увеличить время разработки в некоторых случаях.
2. Изучение и внедрение: Использование TypeScript требует понимания типизации и некоторой дополнительной работы по настройке и интеграции в процесс разработки.
3. Ограничения динамического типирования: TypeScript может требовать более явного описания типов, что может быть непривычным для разработчиков, привыкших к динамической типизации JavaScript.

**Типы в TypeScript?**

TypeScript предоставляет несколько встроенных типов данных, таких как:

Числа: number
Строки: string
Булевы значения: boolean
Массивы: Array<тип> или тип[]
Объекты: object
Функции: Function
Перечисления: enum
Null и undefined: null и undefined
Any: any (тип, который может представлять любой тип данных)
Также TypeScript позволяет создавать пользовательские типы данных с помощью интерфейсов и типовых объявлений.

**Что такое декораторы?**

Декораторы в TypeScript - это специальные функции, которые могут применяться к классам, методам, свойствам и параметрам функций для расширения и изменения их поведения. Декораторы представлены в виде аннотаций, начинающихся с символа @, и могут использоваться для добавления метаданных, применения аспектов, модификации или расширения функциональности классов и компонентов.

**Поддерживает ли TypeScript перегрузку функций?**

TypeScript поддерживает перегрузку функций. Перегрузка функций позволяет определить несколько вариантов сигнатур функции с различными параметрами и типами возвращаемых значений. При вызове функции TypeScript выберет наиболее подходящую сигнатуру функции на основе переданных аргументов.

**Разница между типом (type) и интерфейсом (interface)?**

Разница между типом (type) и интерфейсом (interface) в TypeScript заключается в их назначении и возможностях.

- Тип (type) позволяет определить новый тип данных, который может быть применен к переменным, параметрам функций и другим местам. Он может представлять простые типы данных, объединение типов, пересечение типов и т.д. Типы также могут быть обобщенными (generic).
- Интерфейс (interface) определяет структуру объекта или класса и описывает его свойства, методы и сигнатуры функций. Интерфейсы могут быть реализованы классами и использоваться для обеспечения согласованности типов в различных частях кода.

**Что такое JSX в TypeScript? Какие режимы JSX поддерживает TypeScript?**

JSX в TypeScript представляет собой расширение синтаксиса JavaScript и позволяет писать XML-подобный код в JavaScript/TypeScript файле. JSX обычно используется вместе с библиотеками пользовательских интерфейсов, такими как React, для описания структуры и внешнего вида компонентов.

TypeScript поддерживает два режима JSX:

- preserve: В этом режиме JSX-синтаксис сохраняется в результирующем коде без преобразования в вызовы функций.
- react: В этом режиме JSX-синтаксис преобразуется в вызовы функций React для создания элементов пользовательского интерфейса.

**Что такое директивы с тремя наклонными чертами (Triple-Slash Directives), их типы?**

Директивы с тремя наклонными чертами (Triple-Slash Directives) - это специальные комментарии в начале файла TypeScript, которые предоставляют инструкции для компилятора. Они используются для импорта модулей, указания пути к файлам объявлений (declaration files), настройки компиляции и других задач. Некоторые типы директив:

`/// <reference path="..." />`: Используется для указания пути к файлу объявлений, который должен быть использован компилятором.
`/// <reference types="..." />`: Используется для импорта объявлений типов для сторонних модулей.
`/// <reference lib="..." />`: Используется для включения определенных библиотек в компиляцию.

**Что такое внешние объявления переменных (ambient declaration) в TypeScript?**

Внешние объявления переменных (ambient declaration) в TypeScript позволяют описывать существующие объекты и библиотеки JavaScript без необходимости включения их исходного кода в проект. Они позволяют объявить структуру и типы данных для внешних библиотек, чтобы TypeScript мог предоставить подсказки, проверки типов и другие возможности разработки.

**Разница между абстрактным классом (abstract class) и интерфейсом (interface)?**

Разница между абстрактным классом (abstract class) и интерфейсом (interface) заключается в их назначении и реализации.

- Абстрактный класс - это класс, который не может быть создан напрямую, а может содержать как реализацию методов, так и абстрактные методы. Абстрактные методы объявляются без тела и должны быть реализованы в производных классах. Абстрактный класс может также содержать обычные свойства и методы.
- Интерфейс - это описание структуры и сигнатур функций, которое должны реализовывать классы или объекты. Интерфейсы определяют только контракт, но не предоставляют реализацию. Классы могут реализовывать несколько интерфейсов, в то время как наследование ограничено одним классом.

**Какие элементы ООП поддерживаются в TypeScript?**

TypeScript поддерживает следующие элементы объектно-ориентированного программирования (ООП):

- Классы: позволяют определить структуру, свойства и методы объектов.
- Наследование: классы могут наследовать свойства и методы от других классов.
- Интерфейсы: определяют контракт для классов, которые должны реализовывать определенные методы и свойства.
- Абстрактные классы: классы, которые не могут быть созданы напрямую, но могут содержать реализацию и абстрактные методы.
- Модификаторы доступа: определяют уровень доступа к свойствам и методам класса (public, private, protected).
- Полиморфизм: классы могут использовать наследование и переопределение методов для достижения полиморфизма.
- Инкапсуляция: классы могут инкапсулировать свойства и методы, скрывая их от внешнего доступа.

**Модификаторы доступа в TypeScript?**

Модификаторы доступа в TypeScript определяют уровень доступа к свойствам и методам класса. Типы модификаторов доступа:

- public: доступен из любого места в программе (по умолчанию).
- private: доступен только внутри класса.
- protected: доступен внутри класса и его производных классов.

**Разница между внутренним (Internal Module) и внешним модулями (External Module)?**

Внутренние модули (Internal Modules, сейчас называются пространствами имен - Namespaces) предоставляют средство для организации кода внутри одного файла, позволяя группировать связанные объекты и изолировать их от внешнего кода. Внешние модули (External Modules) представляют собой отдельные файлы, которые экспортируют и импортируют функциональность между различными файлами или проектами.

**Что такое декораторы в TypeScript?**

Декораторы в TypeScript - это специальные функции, которые позволяют изменять поведение классов, методов, свойств и параметров функций. Декораторы применяются с помощью символа @ и могут использоваться для добавления метаданных, применения аспектов, модификации или расширения функциональности объектов.

**Как TypeScript поддерживает необязательные и дефолтные параметры в функции?**

TypeScript поддерживает необязательные параметры в функциях, которые могут быть опущены при вызове функции, и дефолтные значения параметров, которые будут использованы, если аргумент не был передан при вызове функции. Необязательные параметры объявляются с символом ?, а дефолтные значения задаются с помощью =.

**Что такое перечисление (enum)?**

Перечисление (enum) в TypeScript позволяет определить именованный набор числовых или строковых значений. Перечисление предоставляет удобный способ задания и использования ограниченного набора значений в коде.

**Для чего в TypeScript используется NoImplicitAny?**

Опция компилятора noImplicitAny в TypeScript используется для включения/выключения проверки на неявный тип any. Когда эта опция включена, компилятор TypeScript выдаст ошибку, если тип переменной не указан явно и не может быть выведен из контекста.

**Разница между типами “Объединение” (|) и “Пересечение” (&)?**

Разница между типами "Объединение" (|) и "Пересечение" (&) в TypeScript:

Объединение (|): позволяет указывать тип, который может быть одним из нескольких указанных типов. Например, number | string означает, что переменная может быть либо числом, либо строкой.
Пересечение (&): позволяет объединить несколько типов в новый тип, который будет содержать все свойства и методы из всех объединенных типов. Например, A & B означает, что переменная должна быть одновременно типом A и типом B.

**Что такое общие типы (generic) в TypeScript?**

Общие типы (generic) в TypeScript позволяют создавать компоненты и функции, которые могут работать с различными типами данных без явного указания конкретного типа. Общие типы обеспечивают параметризацию типов и повышают переиспользуемость кода.

**Какие области видимости доступны в TypeScript?**

 TypeScript доступны следующие области видимости:

- Область видимости блока: переменные, объявленные внутри блока (например, внутри функции или цикла), видимы только внутри этого блока.
- Область видимости функции: переменные, объявленные внутри функции, видимы только внутри этой функции.
- Область видимости модуля: переменные, объявленные внутри модуля, видимы внутри этого модуля.

**Что такое .map файл, как и зачем его использовать?**

.map файл (source map file) - это файл, создаваемый компилятором TypeScript во время компиляции, который содержит информацию для отображения связи между исходным кодом на TypeScript и соответствующим скомпилированным JavaScript кодом. .map файл позволяет отлаживать исходный TypeScript код в инструментах разработчика, даже если в браузере отображается скомпилированный JavaScript код.

**Можно ли использовать TypeScript в серверной разработке?**

Да, TypeScript можно использовать в серверной разработке. TypeScript может компилироваться в обычный JavaScript, который может выполняться на сервере, таких как Node.js. Это позволяет разрабатывать серверные приложения с использованием преимуществ статической типизации, проверки типов и других возможностей TypeScript.

**Для чего в TypeScript используют ключевое слово declare?**

Ключевое слово declare в TypeScript используется для объявления типов, переменных, функций или модулей, которые уже существуют в других JavaScript-библиотеках или внешних зависимостях. Оно сообщает компилятору TypeScript, что соответствующая сущность будет доступна во время выполнения, но не будет проверяться компилятором.

**Разница между типами void, never и unknown?**

Разница между типами void, never и unknown в TypeScript:

void: тип, используемый для указания, что функция не возвращает значение.
never: тип, используемый для указания, что функция никогда не должна завершаться нормально и не должна возвращать значение.

Например, функция, которая всегда выбрасывает исключение или содержит бесконечный цикл.
unknown: тип, который представляет неизвестное значение. Переменные типа unknown требуют явного приведения типов перед использованием.

**Как вы отлавливаете ошибки в TypeScript коде?**

Ошибки в TypeScript коде могут быть отловлены и исправлены с помощью следующих методов:

- Во время компиляции: TypeScript компилятор выдает ошибки и предупреждения во время компиляции, указывая на проблемные места в коде. Это позволяет обнаружить и исправить ошибки до запуска программы.
- Интегрированные среды разработки (IDE): IDE, такие как Visual Studio Code, предоставляют поддержку TypeScript, включая функции автозаполнения, проверки типов и подсказки ошибок в реальном времени.
- Тестирование: написание и запуск модульных и интеграционных тестов может помочь выявить ошибки в коде и убедиться в его правильной работе.
- Отладка: использование отладчика для выполнения кода по шагам, проверка значений переменных и отслеживание выполнения программы может помочь идентифицировать и исправить ошибки в TypeScript коде.
